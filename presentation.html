Safety by design

Escalation is deterministic, not probabilistic.

Six trigger codes evaluated before any write action. One trigger is enough.

Hard stops (PreToolUse hook)

The hooks/pre_tool_use.py blocks write tools before the agent can act — security queue, frozen accounts, PII exfiltration patterns. The hook fires before the tool. Not a prompt guard.

Slow stops (escalation check)

Six coded triggers in the coordinator. Any one fires → ticket is escalated, urgency set, human paged. No LLM involved in this decision.

Trigger codes

SECURITY_QUEUE P1_PRIORITY CSUITE_REQUESTOR LEGAL_MENTION LOW_CONFIDENCE RETRY_LIMIT

Explicit codes mean consistent behaviour across runs. "When the agent isn't sure" is not a rule.

Defence in depth

Every layer mistrusts the layer above it.

Prompt injection defence

Ticket fields (subject, body, name) are passed as serialised JSON objects — never interpolated into prompt strings. Adversarial content in a ticket cannot escape into the system prompt or alter routing decisions.

Fail-safe account status

When directory lookup fails, account_status defaults to "unknown" — not "active". Unrecognised requestors are escalated, not auto-resolved.

Strict input validation

Pydantic field limits, allowlist checks on queue/priority values, and control-character stripping on every user-supplied string — before anything reaches an LLM prompt or the override store.

Resolver hard blocks

P1 tickets and C-suite requests are blocked from auto-resolution inside resolver_agent.py — not just in the escalation check. Two independent lines of defence for the highest-stakes decisions.
